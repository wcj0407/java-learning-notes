# ##HashMap的底层原理

<img src="D:\zhuomian\java-learning-notes\Java\DAY1 Map\image\HashMap底层原理图.png" style="zoom:50%;" />

当使用hashMap的类的时候，就会创建一个长度为16，加载因子为0.75的数组

当要添加元素的时候，会把元素的键和值存入entry中，然后根据hashcode（）方法和数组的长度来算哈希值

然后就插入到数组中，假如算出来的哈希值指向数组中的位置没有元素就直接插入进去，要是有元素的时候就利用equal方法来比较两个新旧元素，当然比较的只是元素的键而已，和值一点关系都没有，当两个键的值相同时，新的元素entry会直接覆盖旧的entry。要是键的值不相同，要是在JDK8以前，新的元素会替代旧的元素，然后旧的元素会挂在新的元素下面，但是JDK8以后，新的元素会直接挂在旧的元素下面

当数组长度超过64并且挂的元素也超过了八，就会生成红黑树。

--------------

提醒：HashMap底层和HashSet一样都是哈希表

如果键存储的是自定义对象，就要在自定义对象中重写hashcode和equals方法
