# 多线程理解

一个软件的运行就是一个进程，而线程就是进程里面的单位，就是应用软件相互独立，可以同时运行的功能；

进程是操作系统资源分配的基本单位，线程是 CPU 调度的基本单位。

多线程可以提高效率

就是当我们写程序的时候，cpu可以利用不同的时间差来完成多件事情



# 并发

就是在同一个时刻上，有多个指令在单个CPU上交替执行



# 并行

在同一个时刻，有多个指令在多个CPU上同时执行

## 多线程的生命周期

1. 五大状态
新建new Thread () 后，对象创建完成，但未调用 start()；仅堆内存存在线程对象，操作系统无内核线程。
就绪 RUNNABLE（可运行）调用 start() 后：操作系统创建内核线程，进入就绪队列，等待 CPU 调度。
包含两种细分：就绪（等 CPU）、运行（拿到 CPU 正在执行）
阻塞 BLOCKED竞争同步锁synchronized失败，卡在锁队列；其他线程释放锁后回到就绪。
等待 WAITING / 计时等待 TIMED_WAITING
WAITING：无期限等待 wait()/join()，必须其他线程唤醒notify()
TIMED_WAITING：限时等待 sleep(ms)/wait(ms)，时间到自动回到就绪
终止 TERMINATED
run () 方法执行完毕 / 线程异常退出；线程彻底销毁，无法再次 start ()
