# 生产者和消费者

因为线程具有随机性，我们要是想控制线程的执行，我们就得要有一个容器来存放，来控制线程（缓冲区）

## 等待唤醒机制

### 1.消费者等待

当桌子上没有东西的时候就会进入等待状态，如果有东西的话就立马执行，

### 2.生产者等待

先判断桌子上有没有东西，如果没有东西，就立马做，要是有东西就进入等待，

package WAITNOtifyall;

public class Cook extends Thread {
    @Override
    public void run() {
        while (true) {
            synchronized(Dask.lock) {
                if(Dask.count==0) {
                    break;
                } else {
                    //如果桌子上没有食物
                    if (Dask.Foodflag==0) {
                        try {
                            Dask.lock.wait();
                        } catch (InterruptedException e) {
                            throw new RuntimeException(e);
                        }
                    } else {
                        Dask.count--;
                        System.out.println("吃货正在吃面条还能吃"+Dask.count+"碗");
                        Dask.Foodflag=0;
                        Dask.lock.notifyAll();
                    }
            }
            }
        }
    }

}

### wait() vs sleep() — 三个区别

 sleep：抱着锁睡觉，别人进不来
  synchronized (lock) {
      Thread.sleep(5000);  

// 这5秒内，任何其他线程都进不来！
  }

 wait：释放锁，给别人机会
  synchronized (lock) {
      lock.wait(); 

 // 立刻释放锁，别人可以抢
  }

 wait 的本质就是："我先放手，你搞好了叫我。"
 这就是为什么 wait 必须配合 synchronized 用 — 你得先持有锁，才能释放它。
